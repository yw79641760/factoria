# LESSONS - 从错误中学习

> 永远不要重复同样的错误。每次纠正后，更新这个文件。每条 lesson 保持在 2 句话以内。

---

## 2026-03-03：代码清理策略
行过滤比复杂正则表达式更可靠，更易于理解和维护。在关键的清理/转换步骤添加日志以快速定位问题。

## 2026-03-03：模块化架构
模块化代码更易于维护和调试，虽然增加生成时间，但提供更好的代码质量和可扩展性。多轮对话编排支持复杂场景。

## 2026-03-03：IndexedDB 是浏览器 API
IndexedDB 是浏览器原生的，在 Node.js 中不可用。需要统一存储抽象层来处理不同的环境，自动检测并选择合适的实现。

## 2026-03-03：LLM 代码生成策略
不要试图改变 LLM 的训练习惯。接受它生成的 TypeScript 代码，在后处理中清理，这比提示词工程更可靠高效。

## 2026-03-03：Vercel 环境变量
在 Vercel 中配置环境变量至关重要。部署前验证配置，确保所有必需的密钥都已设置，避免运行时错误。

## 2026-03-02：性能 = 用户价值交付能力
不是生成速度，而是需求契合程度。宁可慢而准确，不可快而无用。用户要的是一个好用的 APP，不是一个快速生成的垃圾。

## 2026-03-01：架构决策
模块化架构从模板驱动升级到能力驱动，解决模板边界不清晰的问题，提供更好的可组合性和扩展性，符合 Agent-Native 理念。
